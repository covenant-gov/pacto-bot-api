## Quick start with Docker Compose

The default stack runs the bot and the `pacto-bot-api` daemon in one project.
The NIP-46 bunker and Nostr relay are optional; add the `with-bunker` and/or
`relay` profiles when you need them.

### Default stack

```bash
cd {{bot_id}}
docker compose up --build
```

This starts the daemon and the bot. By default the daemon expects a relay at
`ws://localhost:7000`; point it at your own relay or use the `relay` profile
below.

### Use the NIP-46 bunker

```bash
cd {{bot_id}}
docker compose --profile with-bunker up --build
```

This adds the bunker, Postgres, and Redis services. The daemon can then use a
bot identity with `backend = "bunker_local"` or `"bunker_remote"`. Set
`PACTO_BUNKER_URI` to the bunker's connection URI.

### Use the internal Nostr relay

```bash
cd {{bot_id}}
PACTO_RELAY_URL=ws://nostr-relay:8080 docker compose --profile relay up --build
```

The internal relay is also reachable on the host at `ws://localhost:7000`.

### Full stack (daemon + bot + bunker + relay)

```bash
cd {{bot_id}}
PACTO_RELAY_URL=ws://nostr-relay:8080 docker compose --profile full up --build
```

### Relay-only

```bash
cd {{bot_id}}
docker compose --profile relay up --build
```

### Multi-bot projects

Add more bots to `bots/` and re-run the default stack. Each bot has its own
`Dockerfile` and handler file; `docker-compose.yml` collects all bot services
at the project root. All bots share the daemon container.

```bash
cd {{bot_id}}
docker compose up --build
```

To add the bunker to a multi-bot project:

```bash
docker compose --profile with-bunker up --build
```

To use the internal relay with any stack, set `PACTO_RELAY_URL` before
starting:

```bash
PACTO_RELAY_URL=ws://nostr-relay:8080 docker compose --profile relay up --build
```

## Local development without Docker

1. Install the vendored SDK wheel:
   ```bash
   pip install ./sdk/dist/*.whl
   ```
2. Move into the bot directory and install the bot:
   ```bash
   cd bots/{{bot_id}}
   pip install .
   ```
3. Run the handler:
   ```bash
   python {{bot_id_snake}}.py
   ```

## Rebuilding the SDK wheel

If you modify the SDK source under `sdk/`, rebuild the wheel before installing
or rebuilding containers:

```bash
python -m build --wheel ./sdk
```

## Adding another bot

```bash
pacto-bot-admin scaffold <new-bot-id> --commands hello,help
```

This creates `bots/<new-bot-id>/` and appends the new bot to `pacto-bot-api.toml`
and `docker-compose.yml`.

## Security note

The generated `pacto-bot-api.toml` contains signing material (`nsec`, bunker
URIs, etc.). It is ignored by `.gitignore` and `.dockerignore` by default and
should never be committed to version control or copied into a Docker image.
