## Quick start with Docker Compose

### Full local stack

Starts the bot, the `pacto-bot-api` daemon, and a bunker signer in one shot:

```bash
cd {{bot_id}}
docker compose --profile full up --build
```

The daemon container publishes its Unix socket at `/run/pacto-bot-api.sock` on
the host. The bot container reads that socket, so the path must be writable by
the container or pre-created with ownership matching the daemon's internal
`pacto` user (uid 1000).

### Host-daemon development

Use this when the daemon is already running on the host:

```bash
cd {{bot_id}}
docker compose --profile bot-only up --build
```

The bot container mounts the host socket read-only.

### Multi-bot projects

Add more bots to `bots/` and re-run the appropriate compose profile. Each bot
has its own `Dockerfile` and handler file; `docker-compose.yml` collects all bot
services at the project root.

## Local development without Docker

1. Install the vendored SDK wheel:
   ```bash
   pip install ./sdk/dist/*.whl
   ```
2. Move into the bot directory and install the bot:
   ```bash
   cd bots/{{bot_id}}
   pip install .
   ```
3. Run the handler:
   ```bash
   python {{bot_id_snake}}.py
   ```

## Rebuilding the SDK wheel

If you modify the SDK source under `sdk/`, rebuild the wheel before installing
or rebuilding containers:

```bash
python -m build --wheel ./sdk
```

## Adding another bot

```bash
pacto-bot-admin scaffold <new-bot-id> --commands hello,help
```

This creates `bots/<new-bot-id>/` and appends the new bot to `pacto-bot-api.toml`
and `docker-compose.yml`.
